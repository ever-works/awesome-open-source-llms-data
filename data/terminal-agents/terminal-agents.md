## Overview

Terminal agents represent the most explicit and controllable approach to agentic coding. Tools like OpenCode and Claude CLI run in the terminal, allowing developers to search repos with precise commands, run tests, inspect logs, and iterate with full transparency.

## Use Cases

- Agentic issue-to-patch loops
- Automation workflows
- Open-weight model experiments
- Workflows requiring full control and auditability
- Cost-control with custom routing

## Features

- Explicit command-driven workflow
- Custom model routing per step
- Enforced diff output
- Automatic test execution
- Evaluation harness integration
- Control over runaway loops
- Full auditability of all operations

## Strengths

- Best place for open-weight model deployment
- Full control over model selection, routing, and costs
- Explicit, auditable loops
- Can enforce strict output formats
- Best for power users and complex automation

## Weaknesses

- Requires technical expertise to set up
- More hands-on than autonomous solutions
- Users build more of their own verification loop
- Less polished UX compared to IDE-based solutions
