## Overview

Cursor is an AI-powered IDE that lives where code lives. It provides a repo-indexed view of the codebase, enabling efficient ask → jump to file → edit → diff → run → iterate workflows. It is the leading product for backend engineering tasks.

## Use Cases

- Backend engineering
- Multi-file edits and refactors
- Type-safe code modifications
- Test-driven development loops
- Codebase exploration with AI assistance

## Features

- Repo indexing for contextual code understanding
- Ask mode for chat-style interaction
- Reviewable diffs as part of the normal flow
- Integrated test loops for verification
- Strong repo understanding reduces redundant token usage
- "Done" is legible through diffs and test results

## Strengths

- Context-aware through indexed repo view
- Naturally enforces diffs and test iteration
- Reduced token waste through better context packing
- Clear verification through reviewable outputs
- Strong for backend, types, and refactoring

## Weaknesses

- Not optimized for UI rendering verification
- Full IDE environment may be heavier than lightweight editors for some workflows
